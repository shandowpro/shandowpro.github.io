12 Git & GitHub - Learn About Branching And Merging Concept:

This lesson => Methodology & Practising on Branching Controling , Branch Merging , Branch Pull Request through [CMD] & [Github GUI] by following steps :

General Notes and Facts about Branching , Merging and Branch Pull Request :

  1. Branch Controling :
    • Command line Commands to deal with Branch by [CMD] with following functions :
      1. Displaying all Available Branches with in your current Opened [Local Repo] -> [Accessing the Local Repo Directory on your PC]
      2. Listing all available Branches -> {git branch} || Alias: {git br}
      3. Creating New Branch -> using (2) next Commands :
        • {git branch NewBranchName} => Create new Branch on this Local Repo
        • {git checkout -b NewBranchName} => Create new Branch on this Local Repo & Open it
      4. Switching between current Branches -> {git checkout BranchName}
      5. Rename Branches -> {git branch -m BranchCurrentName BranchNewName}
      6. Delete Branches -> using (2) next Commands :
        • {git branch -d BranchName} => Delete Branch from this Local Repo (only if this Branch has Uploaded/Merged all it's Files)
        • {git checkout -D BranchName} => Force Delete Branch from this Local Repo (even if this Branch has UnUploaded/UnMerged Files)


  2. Branching Merging and Pull Request Methods :
    • [Merging Local Custome Branch with Local Master Branch + Pushing Local Master Branch to Remote Master Branch] (via {CMD} only) , by next steps :
      1. Accessing the Local Repo Directory => {cd c:/Users/Eng-Shadi/Documents/repos}
      2. Listing all available Branches in this Local Directory => {git branch}
      3. Open/Switch Certain Branch (master) => {git checkout master}
      4. Creating a new Branch (Scroll) => {git branch Scroll}
      5. Creating a new Branch (Dev) & Open/Switch to it => {git checkout -b Dev}
      6. Switching to (Scroll) Branch & Open/Switch to it => {git checkout Scroll}
      7. Creating new Files in (Scroll) Branch => {touch File1.txt File2.js}
      8. Adding new created Files in (Scroll) Branch to Staging Area => {git add File1.txt File2.js}
      9. Commiting new created and Added Files in (Scroll) Branch to Local Repo => {git commit -m "Commiting new 2 Files to the current Branch (Scroll) "}
      10. Creating new Branch for testing & open it -> {git checkout -b Dev}
      11. Rename the (Dev) Branch to (DevNew) -> {git branch -m Dev DevNew}
      12. Force Delete the (Dev) Branch -> {git branch -d DevNew}
      13. Switching to the [Master Branch] -> {git checkout master}
      14. Merging the Branch (Scroll) with Master Branch of Local Repo -> {git merge Scroll}
      15. Pushing Master Branch of Local Repo (after merging the Scroll Branch with it) to the Remote Repo -> {git push origin master }

    • [Directly Pushing Custome Branch of Local Repo -> To Remote Master Branch (To create Pull Request)] (via {CMD} & {Github GUI}) , by next steps :
      1. The same previous Steps , untill before Merging step, (but with out merging)
      2. Pushing the Custome Branch [Scroll] in the Local Repo To the Remote Repo => {git push origin Scroll}
      3. Open the (Github website GUI) to find that the site has create a [pull Request] come from the Custom Branch [Scroll] of Local Repo -> with a message with ready Button [Pull Request] -> Click on it
      4. Finishing procedures of [Pull Request] untill complete Confirming and Merging the Pull Request of Pushed Branch [Scroll] Local Repo -> With [Master] Branch of Remote Repo [origin]
