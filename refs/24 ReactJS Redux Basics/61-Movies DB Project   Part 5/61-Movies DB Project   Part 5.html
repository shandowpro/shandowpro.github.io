Vip Notes about {Routering} with using {link} [Requirements-breifly] :

  1. To use the asssined route [in the App.js] from another component => must commit with the following conditons :
    1. in main [App.js] =>
      • The [user component] OR one of it's inner elements => must be included inside the React Routing system [BrowserRouter] & [Routes]
    2. In the [Router user component] =>
      • Must import the {Link} class to be able to use inside this component
      • The imported the {Link} class Must wrap the eleement desired to be acting as Router
      • The assigned {Link} class Must include the defined router path as [TYPICALLY EQUAL] with the assigned in the [App.js] , to be matched with it as hyperantew path


Choosing between using React method {< Link > to=''} & old method of [<a> href=''] ,to allocate and hypernate into the wrapped element :

  1. [React method {< Link to='' > }] => used if the targeted Router is inside my project [of defeied inner router page ]
  2. [Old method of [< a href='' > ] => used if the targeted Router is an external [Not defined inside the project router system]
