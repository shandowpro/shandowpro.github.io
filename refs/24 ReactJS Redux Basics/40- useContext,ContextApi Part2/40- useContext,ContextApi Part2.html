40- useContext,ContextApi Part2 concept :

useContext, ContextApi() advanced concept :

  1. It is able to assigning a more complex value of the context of the context provider function [within the value prop]
  2. Advanced methods to assign values of the [value] props of the contexts , as following :
    • using {useState} Hook => as a dynmaic Assinging method of value of the context provider function [ by single & object value ]
    • using {useRef} Hook => as a dynmaic Assigning and Get value of the context provider function
    • using {useState} & {useRef} Hooks => as a dynmaic Assigning and Get value of the context provider function
  3. Main steps to created each method of advaced methods with assingng the context value , as following :

    a] using {useState} Hook => as a dynmaic Assinging method of value of the context provider function by [Single value] , as following steps :

    • 1] {ContextFile.js} => {Main Context definition file} =>
      // Importing the {createContext} object & {useState} Hook form the reactJS main library :
      import React , {createContext , useState} from 'react' ;

      // Creating the main context [ with using {createContext} object] before the functional Component of [context provider function] :
      const ContextVar = createContext() ;

      // Define the functional Component of [context provider function] {with using (children) prop} + {using the defined state varialbe and function inside the [value] prop} :
      const ContextProviderFunc = ({children}) => {

      // Creating {useState} Hook variable inside the functional component [context provider function] :
      const [stateVar , stateHandler] = useState('value') ;

      // Define the parent state handler function of [defined state varialbe] + using a parameter to change the value of the context by :
      stateHandler(par) ;
      }

      return (
      // Using the defined context variable as an element [with using {provide} object of this element] & [with a {value} prop for this element assigned by [defined object of {State variable} & {Parent handler function}] => to be sent to the importor component] :
      < ContextProvider.Provide value = { {stateVar, stateParentHanlder } } >
      {children}
      < /ContextProviderFunc >
      )
      }

      export {UseContextVar , ContextProviderFunc } ;




    • 2] {App.js} the user component => using the defined and imported {context variable} => importing, custom defining and using the recieved object of useState [from {value} prop of the context function as object of {State Var} and {State function} ] the defined context variable {ContextVar} :
      // Importing the defined and exported Context Provider variable {UseContextVar} to be used/extracted with {useContext} object :
      import {ContextVar} from './component/ContextFile.js' ;

      // Import the {useContext} object from main react Library ;
      import React , {useContext} from 'react' ;

      // Define the main function of [App.js] :
      const App = () => {
      // Define/Extracting a new Custom Context Variable of the imported Context Variable [with using {useContext} importerd object] - inside the user function component , and before the return :
      const {stateVar , stateParentHanlder } = useContext(ContextVar) ;

      return (
      < div >
      // using the defiend {Custom Context State Variable} to display hte current value fo the context :
      {stateVar}
      // using the defiend Context {State parent handler function } to change the value of the context :
      stateParentHanlder('new value of the context') ;
      < /div >
      )
      }

      export default App ;





    b] using {useState} Hook => as a dynmaic Assinging method of value of the context provider function by [object multiple values] , as following steps :

    • 1] {ContextFile.js} => {Main Context definition file} =>
      // Importing the {createContext} object & {useState} Hook form the reactJS main library :
      import React , {createContext , useState} from 'react' ;

      // Creating the main context [ with using {createContext} object] before the functional Component of [context provider function] :
      const ContextVar = createContext() ;

      // Define the functional Component of [context provider function] {with using (children) prop} + {using the defined state varialbe and function inside the [value] prop} :
      const ContextProviderFunc = ({children}) => {

      // Creating {useState} Hook variable inside the functional component [context provider function] :
      const [stateVar , stateHandler] = useState( {id:'value' , name:'value'} ) ;

      // Define the parent state handler function of [defined state variable] + [parameters of used object properties] to change the value of the context by :
      const stateParentHanlder = (id , name) => {
      stateHandler({id:id , name:name}) ;
      OR => stateHandler({id, name}) ; => [in modern JS]
      }

      return (
      // Using the defined context variable as an element [with using {provide} object of this element] & [with a {value} prop for this element assigned by [defined object of {State variable} & {Parent handler function}] => to be sent to the importor component] :
      < ContextProvider.Provide value = { {stateVar, stateParentHanlder } } >
      {children}
      < /ContextProviderFunc.Provider >
      )
      }

      export {UseContextVar , ContextProviderFunc } ;




    • 2] {App.js} the user component => using the defined and imported {context variable} => importing, custom defining and using the recieved object of useState [from {value} prop of the context function as object of {State Var} and {State function} ] the defined context variable {ContextVar} :
      // Importing the defined and exported Context Provider variable {UseContextVar} to be used/extracted with {useContext} object :
      import {ContextVar} from './component/ContextFile.js' ;

      // Import the {useContext} object from main react Library ;
      import React , {useContext} from 'react' ;

      // Define the main function of [App.js] :
      const App = () => {
      // Define/Extracting a new [Custom Context Object(of defined state object)] of the imported Context Variable [with using {useContext} importerd object] - inside the user function component , and before the return :
      const {stateVar , stateParentHanlder} = useContext(ContextVar) ;

      return (
      < div >
      // using the defiend {Custom Context State object properties } to display hte current value fo the context :
      {stateVar.id}
      {stateVar.name}
      // using the defined Context {State parent handler function} to change the value of the context :
      stateParentHandler('new value of the first object property of context variable' , 'new value of the second object property of context variable') ;
      < /div >
      )
      }

      export default App ;





    c] using {useState} & {useRef} Hooks => as a dynmaic Assinging method of value of the context provider function by [single Value] + dynmaic recieving context value method form user by useRef , as following steps :

    • 1] {ContextFile.js} => {Main Context definition file} =>
      // Importing the {createContext} object & {useState} Hook form the reactJS main library :
      import React , {createContext , useState} from 'react' ;

      // Creating the main context [ with using {createContext} object] before the functional Component of [context provider function] :
      const ContextVar = createContext() ;

      // Define the functional Component of [context provider function] {with using (children) prop}
      + {using the defined state variable and function inside the [value] prop} :
      const ContextProviderFunc = ({children}) => {

      // Creating {useState} Hook variable inside the functional component [context provider function] :
      const [stateVar , stateHandler] = useState('value') ;

      // Define the parent state handler function of [defined state varialbe] + using a parameter to change the value of the context by :
      const StateParentHanlder = (par) => {
      stateHandler(par) ;
      }

      return (
      // Using the defined context variable as an element [with using {provide} object of this element] &
      [with a {value} prop for this element assigned by [defined object of {State variable} & {Parent handler function}] => to be sent to the importor component] :
      < ContextProvider.Provide value = { {stateVar, StateParentHanlder } } >
      {children}
      < /ContextProviderFunc.Provide >
      )
      }

      // Exporting both context variable and [Context Fuction Provider] :
      export {UseContextVar , ContextProviderFunc } ;




    • 2] {UseRefExp.js} the user component => including the folowing steps :
      1. using the defined and imported {context variable} to Extract/define a custom variable by the recieved object of useState [from {value} prop of the context function as object of {State Var} and {State function}] the defined context variable {ContextVar}
      2. Define a useRef variable to use it to assign it's element's value as a context value

      // Importing the defined and exported Context Provider variable {UseContextVar} to be used/extracted with {useContext} object :
      import {ContextVar} from './component/ContextFile.js' ;

      // Import the {useContext} object & {useRef} object from main react Library ;
      import React , {useContext , useRef} from 'react' ;

      // Define the main function of [App.js] :
      const App = () => {
      // Define/Extracting a new Custom Context Variable of the imported Context Variable [with using {useContext} importerd object]
      - inside the user function component , and before the return :
      const {stateVar , stateParentHanlder } = useContext(ContextVar) ;

      // Define a Ref varaible => [to use in access to the inserting element and it's value ]
      const elem = useRef() ;

      // Define a function of assinging the Ref value of the input element to the imported context
      const changeContext = (ContextVar) => {
      // using the imported function provider function to change the context according to the value of inserted value of the input of defined Ref variable :
      stateParentHanlder(elem.current.value ) ;
      }

      return (
      < div >
      // using the defined & imported {Custom Context State Variable} to display the current value fo the context :
      {stateVar}
      // usign button's onClick event to change the context value according to inserted value of the ref input element , by using the context provider function :
      < button onClick={changeContext} > Click to activate the the function of change context value < /button >

      < /div >
      )
      }

      export default App ;





  4. Requirements of using this [useContext , createContext ,ContextApi] Hook => (Briefly) :

    1] {main context file} -> Creating/Defining the main context {variable} & {provider function } in the file inside the specified file , which is including :

    • Before hte functional component section :
      1. Importing {createContext} object from the React library
      2. Importing {useState} object from the React library => incase of using state varaible to assign the value of the context
      /
    • Define the {context variable} using the imported {createContext} [with empty parameter ]
  • Define the {context provider function} , including the following properties :
    1. Assign the {children} prop parameter of the {context provider function}
    2. Before the Return :
      1. Define the useState variable with the assinging the value of the context inside as it's parameter
      2. Define the useState parent handler function , with using main handler useState function
    3. Inside the {context provider function}'s Return :
      1. using the defined {context variable} + [Provider] as {start-closes element}
      2. Assign the [value] prop of {Context provider element} with object of { defined useState's variable + state parent handler function } ]
      3. Assign the children parameter inside the {Context provider element} as a direct Js element
  • Export both defined {context provider function} & {context variable}


    • 2] {index.js} -> Recieving/importing the exported {context provider function} & wrapping it over the App main element , which is including :

    • Importing { context provider function } form the main defining context file
    • Using Imported {Context Provider function} to wrapp the main {App} component



    3] {user component file} [App or any child component ] -> using the created/defined {context variable} , which is including :

    • Importing the deinfed { context variable } form the main defining context file
    • Importing the deinfed { useContext} object form the main 'react' library
    • Extracting a new custom context variable from the imported context variable
    • Using/calling the new defined {Custom context variable} inside the functional component , as direct JS element code - between {}

    • {useContext, ContextApi()} => usages and importance :

    1. Used mainly to provide public access to specific data fo certain datatype
    2. Very helpful in large React project
    3. The best solution of the a big problem in large scale React projects , which called => {Prop Drilling}
    4. Can be combined with tyhpes of the hooks funcion to perform a greate and advnaced methodology




    Important Notes :

    • Features and properties of using {ContextApi} in the ReactJS :
      1. Ability for public access to a certain defined data storage of any datatype , from any location in the react project functional component
      2. This methodology is very essential in large reactJS project specially

    • Notes About the main { Define context file} :

      [Context main varaible] => is being extracted from imported { createContext } object from the main 'react'

      [Context provider function] =>

      1. Is being defiend by random name + must using the {children} prop inside to be able to access any user component
      2. It is returning a {context provider element} according to the defiend main context variable [before the define the main context provider function ] -> [by adding the provider object to the main defeind conitext variable ]
      3. In case of using the [useState variable] as the variable of {context provider element}'s [value] prop => Must assign a value inside the state main varaible parameter [inside the brackets
      4. In case of using the [useState variable] as the variable of {context provider element}'s [value] prop => Must define the {State varialbe} and {Handler function} inside this main [context provider function] and before the return
      5. In case of using the [useState variable] => must using the both of state variable and parent handler as object value of the retruned {context provider element}

      Exporting both {context main varaible} & { Context provider function } , without using default keyword


    • Notes About the { main index file } :

      [Importing] => must importing main defined {Context provider function} name form the context file

      The imported main context provider function must my used as wrapper element of the App main element of the application

    • Notes About the { user context component file } :

      [Importing] =>

      1. must importing the main {context defined variable} from the main context file
      2. must import the {useContext } object from the main 'react' library , to use it in extracing a new context varialbe
      3. must import the {useRef} object from the main 'react' library => incase of using the ref variable with the context

      [Extracting a new Context Variable from the Imported & recieved context] :

      • [incase of [using a simple value] in element provider's [value] prop in the main define context file] :
        • [Extracting a new context variable] => Using a normal/single variable to extract a new context
        • The ability to use the new defined and extracted context variable to display the value of the context only

      • [incase of [using the useSate] as a dynamic method to element provider's value prop in the main define context file] :
        • [Extracting a new context variable] => Must using at least one of {state variable} & {state parent handler} as object variable of new extracted context from the main imported context
        • {state variable} within the new extracted context => used to display the value of the context
        • {state parent handler} within the new extracted context =>
        • Once the main context variable is being imported => both of the {state variable} and {parent handler function} are now accessible in this user component , can able to used inside this component

      [Using the new extracted context variable] inside the current component file :

      1. [incase of [using a simple value] in element provider's [value] prop in the main define context file] :
        • The ability to use the new defined and extracted {context variable} to display the value of the context only
        • [incase of using {useRef}] => ability to Assing the new defined and extracted {context variable} wihtin the defined Ref element's value
        • [incase of using {useRef}] => ability to Get the new defined and extracted {context variable} from defined Ref element's input value

      2. [incase of [using the useSate] as a dynamic method to element provider's [value] prop in the main define context file] :
        • The ability to use the {state variable} of new defined and extracted context variable => to display the value of the context [istead of using the main context variable ]
        • The ability to use the {state parent handler} of new defined and extracted context variable => to assign a new value of the context
        • [incase of using {useRef}] => ability to Assing the new defined and extracted {state variable} wihtin the defined Ref element's value
        • [incase of using {useRef}] => ability to Get a new value of the new defined and extracted context using {state parent handler} with using Ref element's input value
