17-Higher Order Functionsconcept :

[Higher Order Functions] concept :

  • It is function with higher priority execution
  • It has (2) main types :
    1. First type : {With functional parameter} =>

      A function include functional parameter [pre-defined callback function] as one of it's parameters , as following :
      // Define a normal function :
      const nFunc = (par0) => {
      // this funtion will return the parameter that will be recieved by the it's user higher funcion :
      return console.log(par0) ;
      }

      const hoFunc = (par1, nFunc) => {
      // Returning the pre-defined callback as (this function parameter) &
      other this parameter as the callback function's parameter :
      return nFunc(par1) ;
      }

      // calling the higher order function [using the defined function as this function parameter ] :
      console.log(hoFunc('par1 value', nFunc) );


    2. Second type : {With functional return} =>

      A function include functional Return [return a complete function] , as following :
      const hoFunc = (par0) => {
      // Returning complete function [arrow function with (2) parameters ] :
      return (par1) => {
      // printing the both of parameters values at once :
      console.log(`${par0} ${par1} `) ;
      }
      }

      // Define a new functional variable to define the main Higher function's return value inside it ->
      to be able to access the inner return function :
      const funcVar = hoFunc('par0Value') ;
      // [Calling the higher order function directly will case an error {anonymous function} ] ...
      // Calling the new functional variable[as a function with inner return function's parameter]
      will access to the inner return of the higher order function :
      console.log(funcVar('par1 value') );

[Higher Order Functions] practical Exps :

  1. Regular Function [printing the value of the caller function {This} ] syntax :

    {Regular function} by using built-in {window} object & {onload} event :

    // Assigning the element innerHTML by the value of it's caller funtion :
    let test = function () {
    document.getElementByID('elemId').innerHTML = this ;
    }

    // Executing the {test} variable constructor funciton by {onload} event of the {window object} :
    window.onload = test;


    {Regular function} by using built-in {button} object & {addEventListner=> click} event :

    // Assigning the element innerHTML by the value of it's caller funtion :
    let test = function () {
    document.getElementByID('btnId').addEventListner('click' , test) ;
    }





  2. {Regular function} => Solving the problem the This [not accessing the nested inner function ] Syntax :

    Creating interval counter application and solving the {this} problem by defining the value of the this in another variable {let} :

    // Defining constructor funtion :
    let test = function () {
    // Defining a variable will be used to as counter starter
    that = this ;
    this.age = 0;
    // using {setIntevral} built-in function to start counting upon the {This} value which assigned with in the another variable value [that]
    setIntevral( function () {
    that.age++ ;
    console.log(that.age) ;
    } , 1000) ;
    }

    // Extracting a new Object from the constructor function -> to operate the method through it :
    let user = new Person() ;




  3. {Arrow function} => {This} attidute and properties Syntax :

    The {This} has the access through the inner nested function :

    // Defining Arrow constructor funtion :
    let test = () => {
    // Defining a variable will be used to as counter starter
    this.age = 0;
    // using {setIntevral} built-in function to start counting upon the {This} value which assigned with in the another variable value [that]
    setIntevral( function () {
    // Using directly the {this} operator [because it refers to the parent function {test} ] :
    this.age++ ;
    console.log(this.age) ;
    } , 1000) ;
    }

    // Extracting a new Object from the constructor function -> to operate the method through it :
    let user = new Person() ;






Important Notes :

  • First type {Higher order function of [with functioal parameter ] } =>

    this function uses a pre-defined function as one of it's parameter [which is considered as callback function]

    this function can uses more than functional parameter and other parameters isndie the main same high order function

    this function can uses a combine of another parameter inside functional parameter [as it's inner parameter's value ]

  • Second type {Higher order function of [with functional complete function] } =>

    this function defines new inner function inside it's return

    Trying to access to the inner funcaional return will cause an Error [anonymous function ]

    this High order function will NOT be able to access to the inner functional Return directly , but by using [functional variable of the higher function's return value ]

    The inner return function can uses paramters of both main [High order function] and it's own parameter in it's inner return

    When calling the defined [Functional Variable of the higher function's return value] -> must assgn the value of it's inner functional return's parameter
