[ 17 Learn Object Oriented PHP - Class Abstraction Part 2] Concept:

=> PHP OOP [Class Abstraction - Part 2] :
  • Main Concepts for the Abstraced Class =>

    Rules Obligated to be followed by other Classes

    Forcing the deveolopers to follow the instructions in the Abstracted Class

  • More Notes and Properties about {Abstract Class} => :
    • The Extended/Inherited Classes will follow the Abstraced Class Methods and functions ,
      EVEN if the Abstraced Class' Methods has been Deleted later
      and EVEN if the Abstract Class ITSELF has Been Deleted later.
  • More Notes and Properties about {Abstract Methods} => :
    • The Abstraced Class Method's Arguments Not Obligatory to be followed by the same Name of Varaible , but can be changed ,
      with Mandatory to be written in the Inherited Function with in the Extended Classes.
    • The Defined Functions/Methods in the Abstracted Class =>Must be in higher Level in Security [Public - Protected - Private] than the Inherited Methods in the Extened Classes

PHP [PHP OOP 17 Learn Object Oriented PHP - Class Abstraction Part 2] Importance & Benefits :

[Abstract Class] => Handy in case of Removing the inner Methods / Functions in the Abstracted Class => All The Exteneded Class's Inherited Methods and Functions will Still running correctly

[Abstract Class] => Handy in case of Removing the Whole the Abstracted Class => All The Exteneded Class's Inherited Methods and Functions will Still running correctly
