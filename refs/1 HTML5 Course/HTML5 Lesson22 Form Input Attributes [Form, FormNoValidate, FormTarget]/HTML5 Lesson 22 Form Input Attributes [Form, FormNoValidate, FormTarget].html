Form => Input Attribute [ < input   form = ""   > ]

Input Attribute [Form] Concept:

=> Input [Form Attribute] is one of the new HTML5 Form's Input Attributes.

=> Input [Form Attribute] is to make a linked Input with other form Away in the page, which will send its data with the form when submiting.

=> Input [Form Attribute] requiring a Form attribute value Equaled with Form's ID Attribute.

=> Input [Form Attribute] can be allocated for more than one Input Tag outside the form with the page.

Main & Features of Input Attribute [Form Attribute] Usages:

=> It mainly designed to Establish a link between the main Form and any other outside Input feild with in page.

=> This Attribute can work with any Type of Input Feilds located any where in page outside the form.

Example:

Filling Main Form Inputs Fields and outer Inputs Fields linked to the main form .

Fulfill all inputs feilds inside and outside the form and Sending it the same form data:


 


<section>
   <h3> Fulfill all inputs feilds inside and outside the form and Sending
     it the same form data: </h3>
   <form   action="process.php"   method="get"   id="MainForm" >

     <label> Enter your Username: <label>
     <input   type="text"  name="username1"  >

     <label> Enter your password: <label>
     <input   type="password"  name="password"  >

     <input   type="sumbit"  value="send Default" >
     <input   type="sumbit"  value="send Other"  
       formaction="other.php" >

   </form>

   <label> Enter your Nickname: </label>
   <input   type="text"  name="nickname"    form="MainForm"  >

   <input   type="sumbit"  value="OutFormSubmit" >
</section>
Input override Attribute [ < input  type="submit"   formNoValidate   > ]

Input Attribute [FormNoValidate] Concept:

=> Input [FormNoValidate Attribute] is one of the new HTML5 Form's Input Attributes.

=> Input [FormNoValidate Attribute] is to override the [NoValidate] property in specific input (type= "Submit"), to allocate this specified submit button with [NoValidate] property.

=> Input [FormNoValidate Attribute] is to activate [NoValidate] property only to this Submit Button Regardless of this property in Main Form.

=> Input [FormNoValidate Attribute] can be allocated for multiple Input fields (Submit Type) form with the page.

=> Input [FormNoValidate Attribute] Not supported by (Safari) browser.

Main & Features of Input Attribute [FormNoValidate Attribute] Usages:

=> It provides the ability to allocate multiple input feild submit type with [NoValidate] property separetly from the main Form, even it was outer the form and linked with it.

Example:

The next form has two different Submit input fields , one by default properties of the main form, and the second Submit is specified with novalidate property (formNoValidate).

Fulfill your data within the form and Send it by (2) different validation property Submit Buttons :





<section>
   <h3> Fulfill your data within the form and Send it by (2) different
     validation property Submit Buttons :
   <form   action="process.php"   method = "get"   name = "FormValidate">

     <label> Enter your Username: </label>
     <input   type = "text"  name="username2">

     <label> Enter your password: </label>
     <input   type = "email"  name="mail2">

     <input   type = "sumbit"  value = "SubmitDefault" >
     <input   type = "sumbit"  value = "SubmitwithnoValidation" 
       formnovalidate >
   </form>
</section>
Input override Attribute [ < input  type="submit"   formTarget=""   > ]

Input Attribute [FormTarget] Concept:

=> Input [FormTarget Attribute] is one of the new HTML5 Form's Input Attributes.

=> Input [FormTarget Attribute] is to determine the [Target] property of specific input (type= "Submit"), to allocate this specified submit button with [Target] property regardless of the same property of the main allocateFrom.

=> Input [FormTarget Attribute] has the same values of Target property of <a> Tag which are:

  • _blank => To Submit the form Input fields in new Web-Page.
  • _self => To Submit the form Input fields in new Web-Page.
  • _top => To Submit the form Input fields in Web-Page.
  • _parent => To Submit the form Input fields in Web-Page.

=> Input [FormTarget Attribute] can be allocated for multiple Input fields (Submit Type) form with the page.

Main & Features of Input Attribute [FormTarget Attribute] Usages:

=> It provides the ability to allocate multiple input feild submit type with [Target] property separatly from the main Form, even it was outer the form and linked with it.

Example:

Using Different types of Target Property of Form's input Submit fields

Submit the form's input fields with different (4) Target property types:








<section>
   <h3> Submit the form's input fields with different (4) Target property       types: </h3>
   <form   action="process.php"   method = "get">

     <label> Enter your Username: </label>
     <input   type = "text"  name="username3" autocomplete>

     <label> Enter your password: </label>
     <input   type = "password"  name="password3" autocomplete >

     <input   type = "sumbit"  value = "SubmitTargetBlank"
        formtarget="_blank">

     <input   type = "sumbit"  value = "SubmitTargetSelf"
        formtarget="_self">

     <input   type = "sumbit"  value = "SubmitTargetTop"
        formtarget="_top">

     <input   type = "sumbit"  value = "SubmitTargetParent"
        formtarget="_parent">
   </form>
</section>
